## Ketentuan & Batasan Pengerjaan

1. Dikerjakan dalam waktu paling lama 96 jam (4 hari kalender). Setelah waktu pengerjaan dinyatakan
    start, maka hitung mundur akan dimulai.

## Tata Cara

1. Soal akan diberikan 30 menit sebelum waktu start dimulai, sehingga peserta memiliki kesempatan
    30 menit untuk mempelajari dan mencari referensi-referensi.
2. Pada saat pengerjaan, perusahaan akan menyiapkan repo github/gitlab yang akan digunakan untuk
    menyimpan hasil pekerjaan.
3. Sebelum jam pengerjaan berakhir, peserta tes harus push/upload script hasil ke repo yang
    disediakan. Pada saat jam berakhir, repo akan di lock sehingga tidak bisa dilakukan push/update
    script oleh peserta
4. Pada saat wawancara/review hasil uji teknis, peserta uji teknis diharuskan mempresentasikan hasil
    pengerjaan soal menggunakan script yang telah di simpan di repo yang disediakan sebelumnya oleh
    perusahaan, dengan cara peserta melakukan pull terlebih dahulu (perusahaan akan membuka lock
    repo sementara).
5. Reviewer akan meminta peserta tes untuk memberikan penjelasan terhadap hasil pengerjaan tes
    dan mungkin meminta peserta test melakukan perubahan pada hasil pengerjaan.
**6. Untuk jawaban soal teori, apabila terindikasi menggunakan AI untuk membantu, maka akan**
    **langsung gugur**

## Spesifikasi Teknis Minimal

1. Node versi LTS
2. Framework vue (nuxt) atau react (next)
3. Database postgresql/mariadb/sqlite


4. Tidak diperbolehkan menggunakan tailwind, silahkan gunakan component based framework seperti
    bootstrap/material/foundation.
5. Dijalankan di container docker

## Requirement/Batasan Teknis Khusus

1. Kemampuan untuk membuat script testing akan mendapatkan pertimbangan khusus
2. Silahkan membuat asumsi apabila ada hal harus dipenuhi namun informasinya belum ada di soal.
    Peserta uji bisa juga menambahkan metadata yang dibutuhkan.

## Material Penilaian (Urut Prioritas)

1. Kesesuaian dengan ketentuan, tata cara, spesifikasi teknis minimal, requirement teknis khusus serta
    kesesuaian dengan hasil yang diminta dalam soal
2. Tingkat bug (semakin sedikit semakin baik)
3. Tingkat keamanan (semakin sedikit celah keamanan semakin baik)
4. Presentasi hasil pengerjaan tes dan kesesuaian dengan perubahan yang diminta pada saat
    presentasi/review hasil pengerjaan tes
5. Kualitas database (relationship, metadata, kemampuan penggunaan function/procedure)
_6. Code Readability_
7. UI/UX (tata letak/layout, penggunaan warna, font, kemudahaan penggunaan aplikasi)
8. Lainnya
